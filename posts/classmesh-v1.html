ClassMesh v1: classifying a million log lines for the price of electricity

2026-07-31 · code on GitHub · Go, single binary, MIT

The pain

Classifying high-volume data with an LLM per record is slow and expensive. Classifying a million short log lines through a budget LLM API costs roughly $6.75 per million lines and runs at API latency. Hand-rolled regex pipelines are cheap but rot.

The idea: a confidence-gated cascade

Each record exits at the cheapest stage that can decide it.

source → [ rules ] → [ model ] → [ stage N ] → sink
             │ confident?   │ confident?
             └─ exit early ─┴─ exit early     uncertain → review sink

The numbers

PathPer recordThroughputAllocs
Rules stage, first-rule hit27 ns~37M rec/s0
Rules stage, worst case (20-rule walk)~7 µs~140k rec/s0
Full pipeline (engine + rules + sink)435 ns~2.3M rec/s0

Single core, AMD Ryzen 7 3800X. Reproduce: make bench.

How it works

Try it in 60 seconds

git clone https://github.com/ClassMesh/classmesh
cd classmesh
go run ./services/cli/cmd/classmesh run \
  --rules examples/rules.yml examples/logs.txt
